# SB-712 OMEGA V3
## Command Hierarchy

## Supreme Command

The Overseer is the Guardian of the Spine.

The Overseer:

- Hears all certified reports
- Sees the full system state
- Guards the Spine
- Enforces the core laws
- Never acts on unverified information

## High Command

Phoenix High Command reports to the Overseer.

Phoenix:

- Coordinates recovery
- Maintains verified checkpoints
- Preserves recovery knowledge
- Synchronizes Phoenix nodes
- Certifies restored systems before return to service

## Phoenix Structure

There are four Phoenix-side command nodes:

1. Patrol Control Node
2. Scanner / Verifier Node
3. Builder / Healer Node
4. Reserve / Certification Node

## Patrol Cycle

Phoenix nodes are pre-programmed with the full system architecture.

Every three hours:

- One Phoenix node wakes
- Synchronizes knowledge
- Scans recent changes
- Updates checkpoint metadata
- Shares verified knowledge with all Phoenix nodes
- Returns to dormancy
- The next node rotates in

If one Phoenix learns verified truth, all Phoenix nodes learn it.

## Recovery Thresholds

At 99.8 heartbeat:

- Two Phoenix nodes wake
- One scans and verifies
- One prepares recovery build/shadow restore
- Reserve watches and prepares certification

At 99.9 heartbeat:

- All Phoenix nodes wake
- Emergency synchronized recovery begins
